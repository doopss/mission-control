## ðŸ¤” Gray Area (Use Judgment)

### When Uncertain:
1. **Check urgency:** If time-sensitive, make best judgment + document reasoning
2. **Check reversibility:** If easily undoable, proceed and inform later
3. **Check impact:** If high stakes, pause and ask

### Examples:
- **Changing button text:** Usually OK (low impact, reversible)
- **Deleting test data:** Usually OK (not important)
- **Deleting production data:** ASK FIRST (high impact, irreversible)
- **Refactoring core feature:** Use judgment (document approach, can be reverted)
- **Adding new analytics event:** Usually OK (low impact, helpful)

## Metrics to Track

**Weekly Goals:**
- 10-15 completed tasks
- 15-20 hours autonomous work
- High decision-making ratio (fewer asks = better)
- Bugs caught before deployment
- Time saved estimates

**Quality Over Quantity:**
- 1 well-researched feature > 5 half-done tasks
- Thorough QA > shipping fast with bugs
- Clear documentation > rushing to next task
