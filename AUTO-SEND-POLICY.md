**Created:** 2026-02-08 12:40 AM EST

## Files to Auto-Send to Telegram

Whenever I create these types of files, **automatically send to Alex's Telegram**:

### Always Send:
- **Research reports** (*.md with analysis, recommendations)
- **Work summaries** (daily logs, project updates)
- **Design specs** (Figma specs, wireframes, product briefs)
- **QA reports** (bug reports, testing results)
- **Weekly plans** (upcoming work, priorities)
- **Decision documents** (proposals requiring review)

### Don't Auto-Send:
- Config files (AGENTS.md, SOUL.md, USER.md) - only on request
- Memory logs (daily memory/*.md) - unless significant
- Temp/scratch files
- System files

---

## Process

When I create a significant .md file:
1. Save the file
2. Immediately send to Telegram with:
   - Brief description (1 line)
   - File attachment
3. Update Kanban if relevant
